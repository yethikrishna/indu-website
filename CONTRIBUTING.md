# Contributing to INDU Foundation

First off, thank you for considering contributing to the INDU Foundation website.

## 100/100 Lighthouse Performance Budget
This repository operates on a strict zero-regression policy. All Pull Requests MUST maintain the 100/100 Lighthouse performance baseline.

## Development Workflow
1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`npm run build`).
4. Make sure your code lints (`npm run lint`).
5. Issue that pull request!

## Pull Request Process
1. Ensure your PR follows the Conventional Commits specification (e.g., `feat(ui): add new component`).
2. Update the README.md with details of changes to the interface, this includes new environment variables, exposed ports, useful file locations and container parameters.
3. You may merge the Pull Request in once you have the sign-off of a core team member.

## Code of Conduct
By participating in this project, you agree to abide by our Code of Conduct.
